# AGENTS.md — Documentation

## Scope
Applies to `docs/` and all subdirectories.

## Mission
Keep goals, architecture, milestones, skills, and mod-author guidance clear enough that new contributors can work independently.

## Folder Responsibilities
- `goals/`: product goals, milestones, release criteria, risk tracking.
- `architecture/`: architecture decision records, diagrams, subsystem contracts, compatibility policies.
- `skills/`: skills matrix, learning paths, domain references, contributor onboarding.

## Goal
Document how to build an original moddable voxel sandbox and how each team can contribute safely without copying proprietary content.

## Skills Needed
- Technical writing.
- Architecture documentation.
- API reference writing for mod authors.
- Test-plan and acceptance-criteria design.
- Contributor onboarding and project planning.

## Rules
- Documentation must distinguish inspiration from copying; require original implementation and assets.
- Every milestone should include measurable acceptance criteria.
- Public API docs should include examples for mod authors.
- Keep docs close to the systems they describe and update them alongside code changes.
